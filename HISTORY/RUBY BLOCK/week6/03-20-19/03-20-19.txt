  Learning Goals:
    Differentiate between commands and queries in software systems
    Explain how splitting methods into commands and queries can help us write better software


  Some methods are designed to change their object

  Command: change the state of the system, while others are meant to return a result without changing it.
  Query: return a result, without changing anything
